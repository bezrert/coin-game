# ADR-001: источник случайности без своей абстракции

## Статус
Принято.

## Контекст
Казино берёт случайность у Chainlink VRF v2.5. Chainlink живёт только в реальной сети
(Sepolia), в тестах его нет — значит нужна точка подмены, чтобы прогонять логику локально
с предсказуемой случайностью.

Один из вариантов (см. §12.F плана) — завести свой интерфейс `IRandomnessSource` с двумя
реализациями: `ChainlinkVRF` и `Mock`.

## Решение
`Casino` наследует `VRFConsumerBaseV2Plus` напрямую. Подмена для тестов выполняется на уровне
**адреса координатора** (`VRFCoordinatorV2_5Mock`), который инжектится в конструктор, а не через
самописный `IRandomnessSource`.

## Обоснование
- Это штатный паттерн самого Chainlink: `VRFCoordinatorV2_5Mock` реализует тот же интерфейс
  координатора, что и боевой; в тестах вызывается `fulfillRandomWordsWithOverride(...)`,
  имитируя колбэк. Подмена адаптера уже встроена в дизайн через адрес координатора.
- Своя обёртка `IRandomnessSource` поверх этого — shallow: индирекция без второго реального
  адаптера. Проваливает deletion-test.
- Один реальный адаптер = гипотетический seam; свой интерфейс оправдан только когда
  commit-reveal станет реальным вторым адаптером (по §7 плана не кодим).

## Последствия
Не переоткрывать «а давайте абстрагируем рандом» без появления commit-reveal. Владелец контракта
наследуется из `ConfirmedOwner` внутри `VRFConsumerBaseV2Plus`; отдельный OpenZeppelin `Ownable`
не подключаем (был бы конфликт двух `owner()`).
